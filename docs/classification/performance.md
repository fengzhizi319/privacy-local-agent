> - **语义大模型 (Layer 3)** 用于处理复杂的非结构化病例图像或长篇文本，推理耗时在秒级，但定级推理和 OCR 能力最强。

## 3. 异步 LLM 接口性能

新增异步接口 `/v1/privacy/classify/table/async` 的提交延迟（不含 LLM 实际推理）：

| 场景 | 提交延迟 | 说明 |
|---|---|---|
| 空表 | < 1 ms | 直接返回 job_id |
| 1k 记录表 | ~2 ms | 序列化与任务入队 |
| 队列满载 | 立即返回 429 | 受 `PRIVACY_ASYNC_MAX_JOBS` 限制 |

异步接口将 Layer 3 推理从同步链路剥离，主链路 QPS 与 Layer 1/2 保持一致；LLM 推理在后台线程执行，调用方通过轮询获取结果。

## 4. 影子模式性能影响

开启 `shadow_mode=True` 后，每张表将执行两次分类（当前版本 + 影子版本）：

| 指标 | 影响 |
|---|---|
| CPU | 翻倍 |
| 延迟 | 同步接口翻倍；异步接口提交延迟不变，后台执行时间翻倍 |
| 内存 | 翻倍（同时保留两份结果） |

建议仅在灰度评估阶段开启影子模式，且优先使用异步接口。

## 5. 复合规则性能影响

复合规则在记录级后处理执行，仅增加字段名正则匹配开销：

| 规则数 | 单记录额外开销 |
|---|---|
| 5 条 | < 0.01 ms |
| 50 条 | ~0.05 ms |
| 500 条 | ~0.5 ms |

建议生产环境复合规则数量控制在 100 条以内。

## 6. SecretFlow 适配器性能

SecretFlow 适配器的主要开销在于将联邦分区转换为 pandas DataFrame：

| 输入类型 | 额外开销 |
|---|---|
| HDataFrame 单 partition | 1 次 `to_pandas` |
| VDataFrame | 按列定位 partition，可能涉及多次 `to_pandas` |
| FedNdarray | 转换为 records 后分类 |

建议尽量避免在 SecretFlow 输入上启用 Layer 3 同步推理。

## 7. 性能测试结论与部署建议

1. **流量分流治理**：
   - 绝大多数普通高频字段（如 ID、手机号、普通白名单字段）在 **Layer 1** 就会被极速拦截返回，QPS 达万级，耗时在微秒级，对网关无任何性能压力。
   - 当遇到半结构化临床文本时，按需开启 **Layer 2 Small-NER** 定级，能够以较低的时间成本完成传染病/基因敏感字段的高级定级与风险预警。
2. **大模型异步编排**：
   - **Layer 3 Qwen2-VL** 的推理时间较长（秒级）。在生产环境中，**禁止直接在同步阻塞的 API 网关链路上对高并发流量全量开启 Layer 3**。
   - 建议将 Layer 3 大模型作为**旁路审计队列**，或在前两层判定为 `confidence < 0.6` 需要深度理解或输入内容为图片/手写体扫描件时，才按需触发以节约算力。
   - 使用 `/v1/privacy/classify/table/async` 提交 LLM 任务，避免阻塞同步链路。
3. **影子模式灰度**：
   - 在规则升级前使用影子模式评估差异率，但注意其对延迟与 CPU 的影响。
   - 大表建议使用异步接口配合影子模式。
4. **复合规则控制**：
   - 复合规则适合在记录级执行，规则数量应控制在合理范围，避免影响高并发性能。

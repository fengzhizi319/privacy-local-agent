# 代理转发与负载均衡网关产品设计 PRD

## 1. 概述

本文档定义 `privacy-local-agent` 代理转发与负载均衡网关（API Gateway & Load Balancer）的产品需求与验收标准。该网关作为统一的请求入口，将 REST 与 gRPC 流量分发到后端多个健康的工作节点，实现水平扩展与高可用。

## 2. 设计目标

- 作为轻量级独立组件（也可与 Agent 运行于同镜像中，仅启动参数不同），透明化后端节点集群。
- 支持横向扩展，能够弹性挂载任意数量的后端工作实例。
- 提供多种负载均衡策略，有效应对请求尖峰，提高整体系统吞吐量。
- 实现故障隔离与自动恢复，通过健康检查动态维护可用节点池。
- 支持分布式共享隐私预算账本，保证多实例场景下预算消耗的全局一致性。

## 3. 用户场景

| 场景 | 说明 |
|---|---|
| 高吞吐数据脱敏 | 网关接收并发批量脱敏请求，按策略分发给多个后端节点。 |
| 差分隐私高频聚合 | 多个客户端高频发起 DP 计算，网关合理分散计算负载。 |
| 节点故障自动容错 | 后端节点异常时，网关健康检查将其剔除，后续流量分发至剩余健康节点。 |

## 4. 功能需求

### 4.1 协议转发与代理能力

| ID | 需求 |
|---|---|
| GW-REST-1 | 透明代理 `/v1/privacy/*` 以及 `/health` 等所有 HTTP 端点。 |
| GW-REST-2 | 采用通用路径匹配转发，避免硬编码接口路径。 |
| GW-REST-3 | 正确透传请求头、查询参数与 Body；返回时透传后端响应头与状态码。 |
| GW-GRPC-1 | 实现 `PrivacyService` 所有 RPC 方法。 |
| GW-GRPC-2 | 使用 gRPC 客户端连接池，动态转发请求并返回响应。 |

### 4.2 负载均衡策略

| ID | 需求 |
|---|---|
| GW-LB-1 | 支持轮询（Round-Robin）。 |
| GW-LB-2 | 支持随机选择（Random）。 |
| GW-LB-3 | 支持最小连接数（Least Connections）。 |
| GW-LB-4 | 支持通过配置切换策略。 |

### 4.3 健康检查与自愈

| ID | 需求 |
|---|---|
| GW-HC-1 | 定时向后端节点发送 REST `/health` 与 gRPC `Health` 检查。 |
| GW-HC-2 | 仅双协议检查均通过才视为健康。 |
| GW-HC-3 | 实时更新可用节点池。 |
| GW-HC-4 | 可用池为空时返回 HTTP 503 / gRPC UNAVAILABLE。 |

### 4.4 配置灵活性

| ID | 需求 |
|---|---|
| GW-CFG-1 | 支持 YAML 配置文件或环境变量。 |
| GW-CFG-2 | 可配置网关监听的 REST 端口与 gRPC 端口。 |
| GW-CFG-3 | 可配置后端节点列表（每个节点含 `http_url` 与 `grpc_address`）。 |
| GW-CFG-4 | 可配置负载均衡策略、健康检查间隔与超时时间。 |

### 4.5 分布式共享预算

| ID | 需求 |
|---|---|
| GW-BUDGET-1 | 支持共享持久化预算账本（如 SQLite）。 |
| GW-BUDGET-2 | 保证 epsilon/delta 扣减全局原子。 |
| GW-BUDGET-3 | 未配置数据库时回退为单机内存模式。 |

## 5. 非功能需求

| 维度 | 要求 |
|---|---|
| 转发延迟 | 网关引入的额外耗时 ≤ 5ms。 |
| 并发模型 | REST 基于 FastAPI 异步机制；gRPC 基于 `grpc.aio`。 |
| 连接复用 | 使用应用级单例 HTTP 客户端连接池与 gRPC 连接池。 |
| 鲁棒性 | 优雅处理连接超时、重试、连接被拒绝等异常。 |
| 模块隔离 | 新增代码位于专用模块，减少对原单机业务逻辑的侵入。 |

## 6. 验收标准

- [ ] REST/gRPC 请求正确转发到后端节点。
- [ ] 三种负载均衡策略单元测试通过。
- [ ] 健康检查动态增删节点测试通过。
- [ ] 分布式共享预算一致性测试通过。
- [ ] 网关转发延迟与异常处理测试通过。

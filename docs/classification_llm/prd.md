# 本地多模态大模型分类分级产品设计 PRD

## 1. 概述

本文档定义 `privacy-local-agent` 第三层分类引擎——本地多模态大模型（VLM）分类网关的产品需求与验收标准。该引擎负责处理病例图片、手写病历图片以及 OCR 识别后的凌乱文本，完成高召回、低漏报的自动定级。

## 2. 设计目标

- 在本地安全域内运行多模态推理服务，接收图片或文本输入，自动提取敏感医疗信息并输出分类分级标签。
- 支持病例图片、手写病历图片与极非结构化文本三类输入的自适应识别。
- 基于预置的 L1~L5 分类矩阵进行零样本语义推理，输出结构化 JSON 结果。
- 保证数据 100% 本地处理，不向外部公网接口发送任何敏感病例数据。
- 适配本地开发环境（如 8GB 显存的 GPU），实现轻量化部署。

## 3. 功能需求

| ID | 需求 |
|---|---|
| LLM-INPUT-1 | 支持本地图片路径、Base64 图片、纯文本三种输入格式。 |
| LLM-OUTPUT-1 | 输出固定 JSON 字典，包含 final_level、sub_category、confidence、reasoning、needs_human_review。 |
| LLM-OUTPUT-2 | 禁止输出自由文本，确保系统可 100% 解析。 |
| LLM-TOOL-1 | 提供一键下载脚本，支持从 ModelScope 或 Hugging Face 下载模型权重。 |
| LLM-TOOL-2 | 支持断点续传与自动保存到指定本地模型目录。 |
| LLM-FALLBACK-1 | 自动检测 CUDA/MPS 硬件与模型文件；不满足条件时优雅降级为规则引擎与 Small-NER。 |

## 4. 非功能需求

| 维度 | 要求 |
|---|---|
| 显存控制 | CUDA 上 FP16 显存占用 ≤ 5.5GB；MPS 上 FP32 内存/显存统一开销 ≤ 6.0GB。 |
| 推理延迟 | 纯文本 ≤ 500ms；图片/手写病历识别与定级 ≤ 2.5 秒。 |
| 隐私安全 | 模型 100% 本地执行，不向外部公网发送敏感数据。 |
| 可降级 | 环境不满足时自动回退，保证核心数据流不中断。 |

## 5. 验收标准

- [ ] 本地图片、Base64 图片、文本输入分类测试通过。
- [ ] JSON 结构化输出解析成功率 100%。
- [ ] CUDA/MPS 硬件加速与降级路径测试通过。
- [ ] 显存占用与推理延迟满足非功能需求。

# 本地轻量级 Small-NER 分类定级产品设计 PRD

## 1. 概述

本文档定义 `privacy-local-agent` 第二层分类引擎——本地轻量级命名实体识别（Small-NER）的产品需求与验收标准。该引擎对半结构化医疗文本进行毫秒级实体抽取，作为规则引擎未命中或置信度较低时的补充层。

## 2. 设计目标

- 精准识别中文医学文本中的核心实体：疾病/症状、药物、手术/操作、解剖部位。
- 提供 ONNX 极速模式与 ModelScope 官方管道模式两种运行方式，适配不同资源与精度需求。
- 与第一层规则引擎和第三层 LLM 协同工作，形成递进式分类漏斗。
- 实现联动定级与升级机制。
- 在极简环境下提供无外部依赖的纯 Python BERT Tokenizer。

## 3. 功能需求

| ID | 需求 |
|---|---|
| NER-API-1 | 实现 `extract(text: str) -> List[Dict[str, Any]]` 接口，输出实体列表。 |
| NER-ENT-1 | 支持识别疾病/症状、药物、手术/操作、解剖部位等医学实体。 |
| NER-MODE-1 | 支持 ONNX 极速模式与 ModelScope 官方管道模式。 |
| NER-UPGRADE-1 | 敏感病种与 PII 同现则升级为 L4。 |
| NER-UPGRADE-2 | 基因相关实体标记为 L5 并触发人工复核。 |
| NER-FALLBACK-1 | ONNX 模型未安装或未下载时，自动降级为 `NoOpSmallNerEngine`，输出警告但不抛异常。 |

## 4. 非功能需求

| 维度 | 要求 |
|---|---|
| 延迟 | 单次推理 ≤ 30ms。 |
| 包体积 | ONNX 模型文件 ≤ 100MB。 |
| 兼容性 | ONNX 模式无 `transformers` 依赖。 |
| 鲁棒性 | 缺失模型时优雅降级，不中断整条分类链路。 |

## 5. 验收标准

- [ ] ONNX 与 ModelScope 模式实体提取测试通过。
- [ ] 联动升级策略（L4/L5）测试通过。
- [ ] 优雅降级路径测试通过。
- [ ] 延迟与模型体积满足非功能需求。

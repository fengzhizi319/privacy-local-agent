# 测试控制台前端（Web）产品需求文档（PRD）

## 1. 概述

Privacy 测试控制台前端是 `privacy-local-agent` 的可视化测试工具，面向开发、QA、产品与售前人员。它把 agent 分散的 REST 接口组织为一个**结构清晰、可交互**的控制台：用户无需记忆路径与手写 curl，即可浏览全部能力、加载示例、编辑请求、查看高亮响应，并进行批量回归。

前端为纯静态 SPA，通过统一的 `/api/*` 契约与后端（Python REST 或 Go gRPC 代理）通信，本身不承载任何计算。

## 2. 设计目标

- **简洁美观**：三栏布局 + 分类卡片，避免功能堆砌在同一长页面。
- **功能完整**：覆盖全部端点的浏览、调试、批量测试与辅助工具（cURL / 历史 / 下载）。
- **双后端一致**：同一界面在 Python REST 与 Go gRPC 两个后端间无缝切换。
- **低门槛**：示例开箱即用，中文描述与提示，非专业人员也能演示。
- **轻量**：无第三方 UI 库，构建产物小、加载快。

## 3. 用户场景

| 场景 | 角色 | 描述 |
|---|---|---|
| 浏览能力 | 产品 / 售前 | 总览页按分类卡片浏览全部功能模块，快速了解 agent 能力 |
| 单接口调试 | 开发 | 侧边栏定位接口 → 编辑请求体 → 发送 → 查看高亮响应与耗时 |
| 复现问题 | 开发 | 复制 cURL 命令到终端，脱离 UI 复现请求 |
| 重发请求 | 开发 / QA | 从请求历史回填之前的请求体，微调后重发 |
| 回归验证 | QA | 批量测试一键跑完某分类或全部接口，查看通过率与失败明细 |
| 双后端对比 | 开发 | 切换 Python REST / Go gRPC 后端，对比同一接口的行为 |
| 结果留存 | QA | 下载响应 JSON 文件，归档测试证据 |

## 4. 功能需求

### 4.1 导航与总览（CW-NAV）

| ID | 需求 |
|---|---|
| CW-NAV-1 | 三栏布局：顶部导航栏 + 左侧导航树 + 右侧主区域 |
| CW-NAV-2 | 总览页以分类卡片网格展示全部功能模块，含图标、描述、接口数量与前 3 个接口预览 |
| CW-NAV-3 | 侧边栏按分类分组，支持搜索（label / path / category）与分组折叠/展开，默认折叠 |
| CW-NAV-4 | 侧边栏顶部提供「接口总览」「批量测试」快捷入口 |
| CW-NAV-5 | 仅 Python 后端可用的端点显示 `REST` 角标 |

### 4.2 端点测试（CW-TEST）

| ID | 需求 |
|---|---|
| CW-TEST-1 | 展示接口信息：method、可编辑路径、分类徽章、中文描述 |
| CW-TEST-2 | 请求体编辑器支持 JSON 格式化与校验 |
| CW-TEST-3 | 发送请求并展示响应：状态码、耗时、语法高亮的 JSON |
| CW-TEST-4 | 支持 `Cmd/Ctrl+Enter` 快捷发送 |
| CW-TEST-5 | 错误时展示可读的错误信息（来自后端 `detail`） |
| CW-TEST-6 | 切换端点时重置请求体/响应，不残留上一端点状态 |

### 4.3 辅助工具（CW-TOOL）

| ID | 需求 |
|---|---|
| CW-TOOL-1 | 一键复制当前请求的 cURL 命令（shell 安全转义） |
| CW-TOOL-2 | 复制响应 JSON 到剪贴板 |
| CW-TOOL-3 | 下载响应 JSON 为文件（按路径命名） |
| CW-TOOL-4 | 请求历史持久化（localStorage），支持回填、单条删除、一键清空 |
| CW-TOOL-5 | 历史以相对时间展示（刚刚 / N 分钟前 / ...） |

### 4.4 批量测试（CW-BATCH）

| ID | 需求 |
|---|---|
| CW-BATCH-1 | 选择分类（或全部）一键顺序测试其下所有接口 |
| CW-BATCH-2 | 汇总展示通过率、成功/失败数量 |
| CW-BATCH-3 | 逐条展示状态、接口、耗时、错误信息 |
| CW-BATCH-4 | 点击结果行跳转到对应端点 |

### 4.5 后端切换与状态（CW-BACKEND）

| ID | 需求 |
|---|---|
| CW-BACKEND-1 | 顶栏提供后端切换器（Python REST / Go gRPC） |
| CW-BACKEND-2 | 默认选中与当前页面同源的后端 |
| CW-BACKEND-3 | 顶栏实时展示 agent 健康状态灯 |
| CW-BACKEND-4 | 后端不可达时展示错误页与重试按钮 |

## 5. 非功能需求

| 类别 | 需求 |
|---|---|
| 性能 | 首屏加载 < 1s（本地）；分组/过滤用 `useMemo` 缓存 |
| 兼容性 | 现代浏览器（Chrome / Edge / Safari / Firefox 最新两个版本） |
| 可维护性 | TypeScript 全量类型化；组件单一职责；代码含详细中文注释 |
| 体积 | 无第三方 UI 库，构建产物 gzip 后 < 200KB |
| 降级 | 剪贴板 / localStorage 不可用时静默降级，不影响核心请求功能 |

## 6. 验收标准

- [ ] `npm run build` 通过（含 `tsc` 类型检查）。
- [ ] 总览页正确展示全部分类卡片，点击可进入对应接口。
- [ ] 侧边栏搜索、折叠/展开、快捷入口工作正常。
- [ ] 端点测试可发送请求并展示高亮响应与耗时；`Cmd/Ctrl+Enter` 可发送。
- [ ] cURL 复制、响应复制/下载、请求历史回填均工作正常。
- [ ] 批量测试返回正确的通过率与逐条结果，可跳转端点。
- [ ] 切换后端后 samples 与 health 正确刷新，状态灯实时反映连通性。
- [ ] 后端停止时展示错误页与重试按钮。
